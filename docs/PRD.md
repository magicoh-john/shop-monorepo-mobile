# PRD — 쇼핑몰 앱 (Shopping Mall)

**문서 버전**: v3.0
**작성일**: 2026-05-31
**상태**: 활성

---

## 1. Overview

### 배경
AI 바이브코딩 방식으로 빠르게 구현하는 풀스택 쇼핑몰 서비스입니다. pnpm 모노레포 기반 위에 Next.js와 Prisma를 활용하여 실제 커머스 플랫폼에 가까운 수준의 기능을 구현합니다.

### 목표
Next.js(App Router), NextAuth, Prisma, PostgreSQL, TanStack Query, Zustand를 풀스택으로 활용하여 상품 탐색, 검색, 위시리스트, 장바구니, 주문 결제, 리뷰, 관리자 백오피스까지 실제 쇼핑몰 사이클을 완성한다.

## 3. Functional Requirements

### 3-1. 공통 및 인증
- NextAuth.js 기반 이메일/비밀번호 로그인 및 회원가입.
- Google OAuth 소셜 로그인.
- proxy.ts를 활용한 권한 보호 (마이페이지는 회원만, 백오피스는 관리자만).

### 3-2. 메인 페이지 (`/`)

홈 화면은 아래 섹션 순서로 구성한다.

```
배너 슬라이더
퀵 카테고리 (DB 대카테고리 5개)
개인 추천상품 — 최근 본 상품 (Zustand recentStore, AI 모듈 연동 전 임시)
신상품        — createdAt 최신순 8개
베스트상품    — 주문 수량 상위 8개
전체상품      — 최근 12개 + 전체보기(/products)
```

- 개인 추천상품: 최근 본 상품이 없으면 섹션 숨김. 향후 AI 모듈 연동 시 이 섹션만 교체.
- 퀵 카테고리 아이콘: DB Category 테이블 대카테고리 기준으로 동적 표시.

### 3-3. 상품 페이지 (`/products`, `/products/[id]`)
- 상품 목록 그리드 제공.
- 카테고리 필터링 및 키워드 검색.
- TanStack Query `useInfiniteQuery` 기반 무한 스크롤 페이지네이션.
- 상세 페이지 UI 구성:
  - 좌측: 상품 이미지
  - 우측: 카테고리명, 상품명, 가격, 재고 → 수량 조절(+/−) → [주문하기] [장바구니 담기] [♡ 찜하기] 버튼
  - 하단 1: 리뷰 & 별점 섹션 (스펙 09에서 구현, 이전에는 플레이스홀더)
  - 하단 2: 상품 상세정보 섹션 (product.description)
- [주문하기]: 선택 수량을 장바구니에 담고 `/checkout`으로 즉시 이동.
- [장바구니 담기]: 선택 수량을 장바구니에 추가, 현재 페이지 유지.
- [찜하기]: 위시리스트 토글 (스펙 08에서 구현, 이전에는 UI만 표시).
- 상품 상세 페이지에 평균 별점 및 리뷰 목록 표시 (스펙 09).

### 3-4. 장바구니 & 위시리스트 (`/cart`)
- Zustand를 활용한 클라이언트 장바구니 상태 관리.
- 수량 조절 및 총 결제 금액 자동 계산.
- Zustand를 활용한 위시리스트 상태 관리 (localStorage 영속성).
- Zustand를 활용한 최근 본 상품 관리.

### 3-5. 주문 및 모의 결제 (`/checkout`)
- 배송지 정보 입력 (React Hook Form + Zod 검증).
- 주문 마스터(Order) 및 주문 상세(OrderItem) DB 트랜잭션 저장.
- 공통 코드 테이블(`system_codes`)을 참조하여 초기 상태를 '결제완료'로 세팅.
- 결제 전 주문 취소 기능.

### 3-6. 마이페이지 (`/mypage`)
- 로그인한 유저의 주문 내역 리스트업.
- 배송 상태값(공통 코드 매핑) UI 표시.
- 구매 상품에 리뷰 & 별점 작성.

### 3-8. 관리자 백오피스 (`/admin`)
- 전체 사용자의 주문 목록 테이블 조회.
- 주문 상태(결제대기 → 배송준비 → 배송중 → 배송완료) 변경.
- 상품 등록·수정·삭제 CRUD (Server Actions + Zod 검증).
- 매출 통계 대시보드.

### 3-7. 카테고리 시스템

- 대카테고리/중카테고리 2단계 계층 구조로 DB 관리 (Category 테이블).
- slug 기반 URL로 카테고리 페이지 접근 (`/categories/[slug]`).
- Header 카테고리 탭과 홈 퀵 카테고리 아이콘은 DB 대카테고리 기준으로 표시.
- 오늘특가/신상품/베스트는 카테고리가 아닌 큐레이션으로 홈 화면 별도 섹션 구성.

### 3-9. 상품 리뷰 & 별점
- 구매 완료된 상품에 한해 리뷰 작성 가능 (구매 인증).
- 별점(1~5) 및 텍스트 리뷰 저장.
- TanStack Query를 활용한 리뷰 목록 캐싱 및 낙관적 업데이트.

## 4. Non-Functional Requirements
- **성능**: TanStack Query 캐싱으로 불필요한 API 호출 최소화. DB 인덱싱을 활용한 조회 속도 최적화.
- **안정성**: Prisma 트랜잭션을 활용하여 주문-상세 간 데이터 정합성 보장.
- **타입 안전성**: `packages/types` 공유 인터페이스, Zod 스키마, Prisma 생성 타입을 활용한 End-to-End Type-Safe 환경 구축.
- **상태 일관성**: 서버 상태는 TanStack Query, 클라이언트 전역 상태는 Zustand, 폼 상태는 React Hook Form으로 역할을 명확히 분리한다.
