# 페이지 명세 템플릿

새 페이지 명세를 작성할 때 이 구조를 그대로 채운다.
모든 컴포넌트/색상/타이포그래피는 `DESIGN.md` 토큰을 참조하며, hex 값이나 px 임의 지정을 직접 쓰지 않는다.

---

## 페이지 목적

(한 줄로 이 페이지가 사용자에게 제공하는 핵심 가치를 서술)

## 섹션 목록 (순서대로)

1. ...
2. ...

## 섹션별 명세

### N. 섹션명

- **레이아웃**
  - 데스크탑: (그리드 컬럼 수, max-width, 좌우 margin — `{spacing.*}` 토큰 사용)
  - 모바일: (컬럼 수, gutter — `{spacing.margin-mobile}` 등)
- **사용 컴포넌트** (`DESIGN.md` > `components` 토큰 참조)
  - 예: `{components.product-card}`, `{typography.headline-lg}`
- **콘텐츠** (필수 — 구조 명세만으로 끝내지 않는다)
  - Figma 원본에서 그대로 가져온 실제 텍스트 문구(헤딩, 서브카피, 버튼 라벨 등)를 빠짐없이 기재한다
  - 실제 사용 이미지(또는 이미지 분위기/내용) 명시
  - 샘플 데이터가 있다면 구체적인 예시(상품명, 가격 등)까지 기재 — "적절히 작성"으로 빈칸을 추정하지 않는다
  - DB 데이터로 대체되는 영역은 "DB 쿼리로 대체, 실제 카피 불필요"라고 명시적으로 적어 구분한다
- **반응형 브레이크포인트**
  - 데스크탑 → 모바일 전환 시 변경되는 속성만 명시 (컬럼 수, 폰트 크기, 간격 등)

(섹션마다 반복)

## 페이지 전용 예외 규칙

- DESIGN.md의 일반 규칙과 다르게 적용되는 부분이 있다면 여기에 명시
- 없으면 "없음"으로 기재
