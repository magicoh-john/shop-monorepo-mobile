# 상품 상세 페이지 명세

## 페이지 목적

상품의 이미지·정보·구매 행동을 한 화면에 모아, 사용자가 옵션을 확인하고 장바구니/주문까지 끊김 없이 이어가도록 돕는다.

## 섹션 목록 (순서대로)

1. 상품 이미지
2. 상품 기본 정보 + 구매 액션
3. 배송/혜택 정보
4. 상세 설명
5. 리뷰
6. 최근 본 상품

## 섹션별 명세

### 1. 상품 이미지

- **레이아웃**
  - 데스크탑: 2열 그리드 좌측 컬럼, `aspect-square`, `{spacing.gutter}` 간격으로 우측 정보 영역과 분리
  - 모바일: 전체 너비, `aspect-square` 유지, 정보 영역 위에 배치
- **사용 컴포넌트**
  - `ProductImage` — 단일 이미지만 표시 (`{rounded.md}` 적용)
- **콘텐츠**
  - DB 데이터로 대체 — `product.imageUrl`. 값이 없으면 "이미지 없음" placeholder(🛍️ 아이콘 + "이미지 없음" 텍스트) 노출
  - ⚠️ **DB 미구현**: 쿠팡/네이버처럼 썸네일 여러 장 갤러리(스와이프/썸네일 리스트)는 `Product` 모델에 다중 이미지 필드가 없어 지원 불가. 이번 명세에서는 **단일 이미지 1장**만 다룬다. 다중 이미지 갤러리는 추후 `ProductImage` 모델 추가가 필요한 별도 작업으로 분리.

### 2. 상품 기본 정보 + 구매 액션

- **레이아웃**
  - 데스크탑: 2열 그리드 우측 컬럼, 세로 `{spacing.stack-md}` 간격
  - 모바일: 이미지 아래, 좌우 `{spacing.margin-mobile}` 패딩
- **사용 컴포넌트**
  - `{typography.ko-label-md}` (카테고리명), `{typography.headline-md}`(상품명, Pretendard로 대체), `{typography.headline-sm}`(가격)
  - `ProductActions` — 수량 선택, 주문하기(`{components.button-primary}`), 장바구니 담기(`{components.button-ghost}`), 찜하기(outline 버튼)
- **콘텐츠**
  - DB 데이터로 대체 — 카테고리명(`product.category.name`, 없으면 미노출), 상품명(`product.name`), 가격(`product.price.toLocaleString()` + "원")
  - 예시: 카테고리 "스킨케어" / 상품명 "수분 진정 토너 200ml" / 가격 "28,000원"
  - 구매 액션 고정 카피: 수량 라벨 "수량", 재고 표시 "재고 {n}개", 버튼 라벨 "주문하기" / "장바구니 담기" / "찜하기"
  - ⚠️ **DB 미구현**: 옵션 선택(사이즈/색상 드롭다운), 할인율+원가 동시 표시(예: ~~35,000원~~ 28,000원 -20%), 평점/리뷰 개수 배지는 `Product` 모델에 옵션·할인·평점 필드가 없어 지원 불가. 본 명세에서는 다루지 않으며, 추후 스키마 확장(예: `ProductOption`, `discountRate`, `avgRating` 필드) 후 별도 명세로 추가한다.
- **반응형 브레이크포인트**
  - 모바일에서는 가격 폰트를 `{typography.headline-sm}` → `{typography.ko-body-lg}`로 축소

### 3. 배송/혜택 정보

- **레이아웃**
  - 데스크탑/모바일 공통: 구매 액션 하단, `{spacing.stack-sm}` 간격의 아이콘+텍스트 1줄 리스트, `outline-variant` 1px 구분선으로 위 섹션과 분리
- **사용 컴포넌트**
  - `{typography.ko-body-md}`, `{colors.on-surface-variant}`
- **콘텐츠**
  - ⚠️ **DB 미구현 — 디자인 시안용 고정 placeholder 텍스트**: 배송비/도착예정일/적립혜택 데이터를 저장하는 모델이 없으므로, 실제 값이 아닌 시안 표시용 고정 문구만 사용한다.
  - placeholder 예시: "무료배송 · 오늘 출고 시 내일 도착 예정" / "구매 시 적립금 2% 지급"
  - 실제 배송/적립 로직이 구현되면 이 섹션 전체를 재검토해야 한다.

### 4. 상세 설명

- **레이아웃**
  - 데스크탑/모바일 공통: 전체 너비, `{spacing.container-max}` 내 본문 폭 제한, 상단 `{spacing.section-gap}` 간격
  - 설명 텍스트 아래 상세 이미지 2~3장을 세로로 나열 (`{spacing.stack-lg}` 간격), 각 이미지 `{rounded.md}` 적용
- **사용 컴포넌트**
  - `{typography.ko-body-lg}`
- **콘텐츠**
  - DB 데이터로 대체 — `product.description`. 값이 없으면 "상세 정보가 없습니다" 고정 텍스트 노출
  - 예시: "은행 추출물과 히알루론산이 함유되어 피부 결을 정돈하고 수분을 채워주는 진정 토너입니다. 민감성 피부도 자극 없이 사용 가능합니다."
  - ⚠️ **DB 미구현 — 디자인 시안용 placeholder**: 상세 이미지 2~3장은 `Product` 모델에 다중 이미지 필드가 없어 실제 데이터가 없다. 디자인 시안에서는 placeholder 이미지로 노출하되, 실제 구현 시 `description` 필드 확장(예: 마크다운/HTML 본문에 이미지 삽입) 또는 별도 `ProductDetailImage` 모델 추가가 필요하다.
  - ⚠️ **DB 미구현**: 쿠팡/네이버의 "상세 설명 탭"(상품정보/리뷰/문의 탭 전환 UI)은 지원하지 않는다. 탭 없이 설명 텍스트 + 이미지를 한 섹션으로 단순 노출한다.

### 5. 리뷰

- **레이아웃**
  - 데스크탑/모바일 공통: 상세 설명 하단, `{spacing.section-gap}` 간격
- **사용 컴포넌트**
  - `{typography.ko-body-md}`, `{colors.on-surface-variant}`
- **콘텐츠**
  - 고정 카피 (현재 코드와 동일) — "아직 리뷰가 없습니다"
  - ⚠️ **DB 미구현**: `Review` 모델 자체가 스키마에 존재하지 않는다. 평점 분포, 리뷰 목록, 리뷰 작성 폼 등 쿠팡/네이버식 리뷰 섹션은 본 명세에서 다루지 않으며, 추후 `Review` 모델 추가 후 별도 명세로 작성한다. 현재는 고정 안내 문구만 유지한다.

### 6. 최근 본 상품

- **레이아웃**
  - 데스크탑: 4열 그리드, `{spacing.container-max}` 폭 제한
  - 모바일: 2열 그리드, `{spacing.margin-mobile}` 좌우 패딩
- **사용 컴포넌트**
  - `RecentProducts` (Zustand `recentStore` 기반, 클라이언트 전역상태 — DB 아님)
  - `{components.product-card}`, `{components.product-card-name}`, `{components.product-card-price}`
- **콘텐츠**
  - 클라이언트 상태로 대체 — 최근 조회한 상품 최대 8개. 카테고리명(optional), 상품명, 가격
  - **노출 조건**: 최근 본 상품이 0개이면 섹션 전체를 렌더링하지 않는다 (현재 코드와 동일, `items.length === 0` → null)

## 페이지 전용 예외 규칙

- 옵션 선택, 할인가 표시, 평점/리뷰 통계는 DB 스키마 미구현으로 본 명세에서 제외. 위 각 섹션에 ⚠️ 표시로 명시함.
- 배송/혜택 정보 섹션은 실데이터가 아닌 디자인 시안용 고정 placeholder임을 별도로 표기함.
- 품절 상태(`stock === 0`)는 구매 액션 섹션의 "노출 조건"으로만 처리 — 주문/장바구니 버튼 비활성화 + 라벨 "품절"로 교체, 기본 화면 구성에는 포함하지 않음.
